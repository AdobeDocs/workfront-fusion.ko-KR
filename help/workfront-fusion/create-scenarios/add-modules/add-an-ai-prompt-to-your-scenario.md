# 시나리오에 AI 프롬프트 추가

대형 언어 모델(LLM)과 결합된 모델 컨텍스트 프로토콜(MCP)을 사용하여 시나리오에 AI 프롬프트를 포함할 수 있습니다. MCP 에이전트 모듈에서 이를 구성함으로써 인공 지능을 이용하여 효율적이고 안전하며 유연한 워크플로를 설정할 수 있다.

>[!NOTE]
>
>이 기능은 AI를 사용하여 시나리오에 모듈을 추가하는 기능과 별개입니다.
>
>AI를 사용하는 모듈을 추가하는 방법에 대한 자세한 내용은 [AI를 사용하여 시나리오 세그먼트 생성](/help/workfront-fusion/create-scenarios/add-modules/add-a-module-with-ai.md)을 참조하십시오.

## 모델 컨텍스트 프로토콜 개요

모델 컨텍스트 프로토콜(MCP)은 AI 언어 모델을 다른 애플리케이션과 안전하게 연결하는 방법입니다. AI 모델이 애플리케이션에 액세스할 수 있도록 MCP 서버를 구성합니다. 그런 다음 AI 모델에 프롬프트를 보내고 애플리케이션에서 정보를 반환할 수 있습니다.

예를 들어 AI 모델을 Gmail과 연결하도록 MCP 서버를 구성할 수 있습니다. 큰 언어 모델로 &quot;Gmail에서 최근 5개의 이메일 제공&quot;이라는 프롬프트를 보낼 때 이 프롬프트는 해당 프롬프트를 구문 분석하여 Gmail MPC 서버로 전송하며, 이 서버는 Gmail에 액세스하여 이메일을 반환할 수 있습니다.

MCP 에이전트 모듈을 사용하면 언어 모델과 MCP 서버를 사용하여 사용자 프롬프트를 처리할 수 있습니다.

## Fusion 시나리오에서 MCP 사용의 이점

시나리오에서 MCP를 사용하면 다음과 같은 이점이 있습니다.

* 시나리오에서 MCP를 사용하면 모든 상황과 가능한 행동의 변형을 통해 생각할 필요가 줄어듭니다. 프롬프트를 사용하면 이러한 변형을 설명하기 위해 regex나 구문 분석 데이터를 사용할 필요가 없습니다.
* 이전 모듈에서 매핑된 요소나 매핑 패널의 다른 함수를 사용하여 프롬프트에 컨텍스트를 제공할 수 있습니다.
* 프롬프트에서 추론을 사용하고 사용 가능한 컨텍스트에서 최상의 작업 과정을 선택할 수 있으므로 프롬프트를 사용하면 특정 모듈로 시나리오를 빌드하는 것보다 효율적이고 유연하게 사용할 수 있습니다.
* 모듈이 연결될 수 있는 MCP 서버를 구성하므로 해당 서버의 보안을 제어하고 보안이 조직의 요구 사항에 부합하는지 확인할 수 있습니다.

>[!NOTE]
>
>사용 가능한 기능은 MCP 서버에서 사용할 수 있는 도구에 따라 다르며 Fusion에서 제어되지 않습니다.

## 시나리오에 AI 프롬프트 추가

MCP 에이전트 모듈을 사용하여 시나리오에 AI 프롬프트를 추가할 수 있습니다.

자세한 내용은 [MCP 에이전트 모듈](/help/workfront-fusion/references/apps-and-modules/tools-and-transformers/model-context-protocol-mcp-connector.md)을 참조하세요.
