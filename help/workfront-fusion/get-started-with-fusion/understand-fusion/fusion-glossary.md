# Adobe Workfront Fusion 용어

다음 용어집은 Adobe Workfront Fusion의 몇 가지 일반적인 용어를 설명합니다.


<table style="table-layout:auto">
 <col> 
 <col> 
 <tbody> 
  <tr> 
   <td role="rowheader"> <p>액션</p> </td> 
   <td>선택한 앱이나 서비스에서 데이터를 읽거나 쓰는 등의 작업을 수행할 수 있는 모듈입니다.</td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>집계기</p> </td> 
   <td> <p>여러 번들(여러 데이터 컬렉션)을 하나의 번들로 병합하는 모듈 유형입니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/references/modules/aggregator-module.md" class="MCXref xref">집계기 모듈</a>을 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader">API</td> 
   <td>애플리케이션 프로그래밍 인터페이스(API)는 애플리케이션과 서비스가 서로 통신할 수 있는 방법입니다. Fusion은 API를 사용하여 연결 중인 애플리케이션과 통신합니다. 각 애플리케이션에는 별도의 API가 있습니다. </td> 
  </tr> 
  <tr> 
   <td role="rowheader">API 키</td> 
   <td>소프트웨어의 API를 호출하는 사용자, 개발자 또는 프로그램을 식별하는 고유 코드로, 인증에 사용됩니다. Fusion 모듈은 API를 연결하여 작동하기 때문에 API 키가 필요한 경우가 있습니다. API 키는 필요한 앱에 의해 배포됩니다. 예를 들어, Fusion을 Adobe Lightroom에 연결하기 위해 API 키가 필요하다면 Adobe Lightroom 계정을 통해 요청합니다.</td> 
  </tr> 
  <tr> 
   <td role="rowheader">앱 또는 서비스</td> 
   <td> <p>소프트웨어 애플리케이션. Fusion은 해당 애플리케이션에 대한 전용 커넥터가 없어도 대부분의 애플리케이션에 연결할 수 있습니다.</p> <p>앱은 반복기나 집계기와 같이 데이터를 조작하는 특수 함수가 될 수도 있습니다. </p> <p>서비스는 웹 API, 웹 페이지, 다양한 유형의 서버(FTP, SMTP, IMAP) 등을 포함할 수 있는 데이터 소스입니다. </p>  </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>번들</p> </td> 
   <td> <p>번들은 모듈에서 반환하거나 수신되는 기본 데이터 단위입니다. 예를 들어, 세 개의 레코드를 반환하는 검색 모듈은 각 레코드마다 하나씩 세 개의 데이터 번들을 출력합니다. 번들은 항목으로 구성됩니다.</p> </td> 
  </tr> 
  <tr>
   <td role="rowheader"> <p>연결</p> </td> 
   <td> <p>연결은 주어진 서비스에 연결하기 위한 자격 증명 세트를 나타냅니다. 어떤 모듈 내부에서도 연결을 구성할 수 있으며, 그 연결을 다른 어떤 모듈에서도 사용할 수 있습니다. 모든 모듈은 Fusion이 해당 자격 증명을 사용하여 모듈이 필요로 하는 정보에 액세스할 수 있도록 연결을 선택해야 합니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/get-started-with-fusion/understand-fusion/connection-overview.md" class="MCXref xref">연결 개요</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader">커넥터</td> 
   <td>커넥터는 특정 애플리케이션을 위한 모듈 세트입니다. Workfront Fusion은 Workfront, Salesforce, Jira와 같은 많은 일반적인 업무 애플리케이션에 커넥터를 제공합니다.</td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>주기</p> </td> 
   <td> <p>주기는 시나리오 실행의 두 단계, 즉 운영과 커밋으로 구성됩니다. 시나리오는 하나 이상의 주기로 구성될 수 있습니다. 자세한 내용은 <a href="/help/workfront-fusion/references/scenarios/scenario-execution-cycles-phases.md" class="MCXref xref">시나리오 실행, 주기 및 단계</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>데이터 저장소</p> </td> 
   <td> <p>데이터 저장소는 시나리오의 데이터를 저장하거나 개별 시나리오 또는 시나리오 실행 간에 데이터를 전송할 수 있습니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/create-scenarios/map-data/data-stores.md" class="MCXref xref">데이터 저장소</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>필터</p> </td> 
   <td> <p> 필터는 두 모듈 사이에 적용할 수 있으며, 특정 기준에 맞는 번들로만 작업할 수 있습니다. 다양한 필터를 적용할 수 있습니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/create-scenarios/add-modules/add-a-filter-to-a-scenario.md" class="MCXref xref">시나리오에 필터 추가</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>ID </p> </td> 
   <td> <p>번들을 고유하게 식별하는 데 사용되는 이름입니다. ID는 일반적으로 주어진 서비스에서 업데이트하거나 삭제할 번들을 구분하는 데 사용됩니다. ID는 이전 모듈의 출력에서 매핑할 수 있습니다.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>항목</p> </td> 
   <td> <p>번들의 일부입니다. 번들은 여러 개의 항목으로 구성될 수 있습니다. 항목에는 텍스트, 숫자, 부울(예/아니오), 날짜, 시간, 버퍼(이진 데이터), 컬렉션, 선택 메뉴, 배열, 유효성 검사 등 여러 가지 유형이 있습니다.</p><p> 자세한 내용은 <a href="/help/workfront-fusion/references/mapping-panel/data-types/item-data-types.md" class="MCXref xref">항목 데이터 유형</a>을 참조하십시오.</p> </td> 
  </tr>
  <tr> 
   <td role="rowheader"> <p>반복기</p> </td> 
   <td> <p>하나의 데이터 번들(데이터 컬렉션)을 가져와서 별도의 번들로 나눌 수 있는 모듈 유형입니다. 이러한 번들은 이후 모듈에서 개별적으로 처리할 수 있습니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/references/modules/iterator-module.md" class="MCXref xref">[!UICONTROL 반복기] 모듈</a>을 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>모듈</p> </td> 
   <td> <p>관련 앱이나 서비스 내에서 레코드를 생성하는 등의 기능을 수행하는 시나리오 내의 단일 단계입니다.</p> <p>각 앱이나 서비스에는 요청에 대한 응답 방식을 정의하는 다양한 모듈이 있습니다.</p>  <p> <img src="assets/module.png"> </p> <p>자세한 내용은 <a href="/help/workfront-fusion/get-started-with-fusion/understand-fusion/module-overview.md" class="MCXref xref">모듈 개요</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>작업</p> </td> 
   <td> <p>모듈이 수행하는 작업, 예를 들어 레코드를 검색하거나 파일을 업로드하는 작업입니다.</p><p>자세한 내용은 <a href="/help/workfront-fusion/set-up-and-manage-workfront-fusion/licensing-operations-overview/operations-in-workfront-fusion.md" class="MCXref xref">작업</a>을 참조하십시오.</p>
  </tr> 
  <tr> 
   <td role="rowheader">공개/비공개 키</td> 
   <td>공개 및 비공개 키는 데이터를 암호화하고 해독하는 데 사용됩니다. 공개 키는 배포할 수 있으며, 공개 키를 가진 사람이라면 누구나 데이터를 암호화할 수 있지만, 비공개 키만 해독할 수 있습니다. 마찬가지로, 비공개 키를 가진 사용자는 공개 키를 가진 누구나 해독할 수 있는 데이터를 암호화할 수 있습니다. 비공개 키 암호화는 데이터가 비공개 키 소유자로부터 나왔음을 보장하며, 데이터 소스의 유효성 검사 역할을 합니다.</td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>라우터</p> </td> 
   <td>라우터를 사용하면 시나리오에 데이터를 복제하거나 새로운 경로를 추가할 수 있으므로 데이터를 다시 라우팅하고 다양한 데이터 그룹을 개별적으로 처리할 수 있습니다.</p><p> 자세한 내용은 <a href="/help/workfront-fusion/create-scenarios/add-modules/router-module.md" class="MCXref xref">[!UICONTROL 라우터] 모듈</a>을 참조하십시오.</td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>시나리오</p> </td> 
   <td> <p>사용자가 만든 일련의 자동화된 단계로, 각 모듈에 의해 표현되고 수행됩니다. 시나리오의 목적은 데이터를 이동하고 조작하는 것입니다.</p> <p> <img src="assets/entire-scenario-blank.png" style="width: 350;height: 178;"> </p> <p> 자세한 내용은 <a href="/help/workfront-fusion/get-started-with-fusion/understand-fusion/scenario-overview.md" class="MCXref xref">시나리오 개요</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>시나리오 세그먼트</p> </td> 
   <td> <p> 시나리오 세그먼트는 동일한 애플리케이션에 연결되는 일련의 모듈로 구성된 시나리오의 한 섹션입니다. 시나리오 세그먼트는 종종 애플리케이션에서 짧은 워크플로를 나타냅니다.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>트리거</p> </td> 
   <td> <p>트리거는 새로운 데이터와 업데이트된 데이터를 모니터링하고, 모듈에 구성된 특정 조건이 적용될 때 시나리오를 시작하는 일종의 모듈입니다. 트리거는 일정에 따라 시나리오를 시작하도록 구성하거나(폴링) 데이터 변경이 발생할 때마다(인스턴트 트리거 또는 웹후크) 구성할 수 있습니다.</p> <p>자세한 내용은 모듈 개요 문서에서 <a href="/help/workfront-fusion/get-started-with-fusion/understand-fusion/module-overview.md" class="MCXref xref">트리거</a>를 참조하십시오.</p> </td> 
  </tr> 
  <tr> 
   <td role="rowheader"> <p>웹후크</p> </td> 
   <td> <p>새 번들을 사용할 수 있는 즉시 시나리오를 실행할 수 있는 특별한 유형의 트리거입니다. </p><p>자세한 내용은 <a href="/help/workfront-fusion/references/modules/webhooks-reference.md" class="MCXref xref">인스턴트 트리거(웹후크)</a>를 참조하십시오.</p> </td> 
  </tr>
