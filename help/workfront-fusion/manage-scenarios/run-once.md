# 한 번 실행을 사용하여 시나리오 테스트

시나리오를 만들거나, 업데이트하거나, 세분화할 때 &quot;한 번 실행&quot; 버튼을 사용하여 요청 시 시나리오를 트리거할 수 있습니다. 이렇게 하면 특정 이벤트나 폴링 간격과 같은 트리거 논리를 기다리지 않고 시나리오를 테스트할 수 있습니다.

시나리오를 한 번 실행하여 데이터 구조를 구성할 수도 있습니다

>[!TIP]
>
>시나리오를 만들고 세분화할 때 자주 테스트하는 것이 좋습니다.

## 한 번 실행 기능 사용

한 번 실행 기능은 시나리오 편집기에 있습니다.

1. 왼쪽 패널의 **[!UICONTROL 시나리오]** 탭을 클릭합니다.
1. 시나리오 채점 전문가를 실행할 시나리오를 선택합니다.
1. 시나리오의 아무 곳이나 클릭하여 시나리오 편집기를 입력합니다.
1. 시나리오 편집기의 왼쪽 아래에서 **한 번 실행**&#x200B;을 클릭합니다.
1. (조건부) 시나리오가 웹후크에 의해 트리거되거나 하위 시나리오인 경우 테스트 실행에 대한 입력을 선택합니다.

   * **외부 Webhook 호출을 기다립니다**: 시나리오가 들어오는 Webhook 호출을 수신하기 시작합니다. 웹후크를 제공하려면 외부 시스템에서 웹후크를 트리거해야 합니다.

     예를 들어 웹후크가 Workfront에서 새 요청을 수신하는 경우 Workfront에서 새 요청을 만들어 &quot;한 번 실행&quot; 기능을 완료하도록 트리거해야 합니다.

     >[!NOTE]
     >
     >이 옵션은 웹후크 시나리오에만 사용할 수 있습니다.

   * **이전 실행 입력 사용**: 이전 실행을 선택해야 합니다. 한 번 실행 실행에서는 선택한 실행을 트리거한 입력 데이터를 사용합니다.

     이전 실행을 선택하려면 실행을 선택하고 **실행**&#x200B;을 클릭하세요.

     실행을 선택하기 전에 실행에서 입력 데이터를 검사하려면 해당 실행에 대해 **입력 미리 보기**&#x200B;를 클릭합니다.

     >[!NOTE]
     >
     >이 옵션을 사용하려면 시나리오가 이전 실행을 완료해야 합니다.

   * **수동으로 입력**: 시나리오 입력에 대한 웹후크 페이로드를 제공해야 합니다. JSON 형식이어야 합니다.

     입력을 제공하려면 **Webhook 페이로드** 상자에 텍스트를 입력하고 **실행**&#x200B;을 클릭합니다.
